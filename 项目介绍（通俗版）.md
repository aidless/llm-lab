# llm-lab 是什么？—— 一份给所有人（包括奶奶）看的说明书

> 本文档用最通俗的大白话介绍这个项目。不需要懂编程，也能看懂。

---

## 一、一句话说明白

**llm-lab 是一个“帮你对 AI 进行考试和打分”的小工具。**

你给它一个任务（比如“写一首关于猫的诗”），它就会：
1. 安排 AI 去干活；
2. 检查 AI 干得好不好；
3. 把过程和结果记下来，给你一个“及格 / 不及格”的判定。

就这么简单。

---

## 二、用一个生活比喻来理解

把它想象成**学校里的一次“随堂测验”**：

- **你（用户）** 就是出考题的老师：“请写一首关于春天的诗。”
- **AI 模型**（比如 GPT、Claude） 就是**参加考试的学生**。
- **出题和安排流程的部分（Planner）** 是“监考老师”，负责把考题变成一张标准试卷。
- **检查答案的部分（Verifier）** 是“阅卷老师”，看看学生写得对不对、合不合格。
- **记录本（Tracer / 数据库）** 是**成绩单**，把每次考试谁考了、考得怎样、得了多少分，都记下来。

这个工具的作用，就是**把“出题 → 考试 → 阅卷 → 记成绩”这一整套流程自动化**，而且可以同时让好几个“学生”（不同的 AI 模型）参加同一场考试，方便你比较谁更厉害。

---

## 三、它能帮你做什么？

| 你想要做的事 | 这个工具能做的 |
|---|---|
| 让某个 AI 帮我完成一个任务，并确认它做对了 | ✅ 用 `run` 命令 |
| 比较两个不同的 AI，看谁做得更好 | ✅ 用 `compare` 命令（A/B 对打） |
| 一下子让好几个 AI 同时做同一件事 | ✅ 用 `batch`（批量）功能 |
| 在网页上点一点就能用，不用敲命令 | ✅ 用 `serve` 打开网页界面 |
| 回头查看“上次那次考试结果怎么样” | ✅ 用 `history`（历史记录） |
| 把结果导出来给别人看 | ✅ 用 `export`（导出成文件） |

## 四、三种最常用的用法（大白话版）

### 1. 单独考一个 AI：`run`

就像你说：“让 GPT 写一首诗，看看写得怎么样。”

```bash
llm-lab run "写一首关于 Rust 编程语言的俳句"
```

工具会：让 AI 写诗 → 检查这首诗合不合格 → 告诉你“通过 / 没通过”，并把过程记下来。

### 2. 让两个 AI 比一比：`compare`

就像你说：“让 GPT 和 Claude 写同一首诗，看看谁写得更好。”

```bash
llm-lab compare "写一首关于 Rust 编程语言的俳句" --model-a gpt-4o --model-b claude-3-5-sonnet
```

工具会：让两个 AI 都写 → 分别阅卷 → 生成一份“对比报告”，告诉你谁更胜一筹、各自花了多少钱。

### 3. 打开网页点一点：`serve`

就像打开一个 App，不用记命令，在网页上填任务、点按钮就行。

```bash
llm-lab serve
```

然后打开浏览器访问 `http://localhost:8123` 就能用了。

## 五、它背后是怎么干活的？（简单流程图）

```
你下达任务
   │
   ▼
【安排】把任务变成一张标准“试卷”
   │
   ▼
【考试】让 AI（学生）去完成任务
   │
   ▼
【阅卷】检查 AI 的回答对不对、好不好
   │
   ▼
【记成绩】把过程、花费、判定（及格/不及格）写进“成绩单”
   │
   ▼
【给你看】在屏幕或网页上展示结果
```

> 小知识：这一次我们重点完善的就是最后一步“**记成绩**”——
> 以前工具会忘记把“及格还是不及格”这一栏写进成绩单；
> 现在无论是用命令、用网页、还是让好几个 AI 一起考，
> 每一场考试的“判定结果”都会被清清楚楚地记下来，不会丢。

## 六、怎么把它跑起来？（给想动手的人）

> 这一节需要一点点电脑基础。只想了解是什么的人可以跳过。

**第 1 步：安装**
```bash
pip install -e ".[dev]"
```

**第 2 步：准备“钥匙”**
把项目里的 `.env.example` 文件复制一份，改名为 `.env`，在里面填上你的 AI 账号钥匙（叫 `LLM_API_KEY`）。
> 就像进图书馆要办一张借书卡，这把“钥匙”就是 AI 服务的借书卡。

**第 3 步：开跑**
```bash
llm-lab run "写一首关于猫的诗"
```

**第 4 步（可选）：用网页**
```bash
llm-lab serve
```
浏览器打开 `http://localhost:8123`。

## 七、奶奶可能会问的几个问题

**问：AI 不是已经很聪明了吗，为什么还要“考试”？**
答：AI 有时候会一本正经地胡说八道（行内叫“幻觉”）。这个工具就是帮你在让 AI 干活之后，**再检查一遍它有没有做对**，避免被它带偏。

**问：为什么要比较不同的 AI？**
答：不同的 AI 各有长短。有的写诗好，有的算数强。比一比，你才知道该用哪一个，还能看看谁更省钱。

**问：“及格 / 不及格”是谁定的？**
答：工具里有个“阅卷老师”（Verifier），可以按规则检查，也可以用一个更聪明的自动评判（DeepEval）来打分。分数过了线就算及格。

**问：我的数据会跑出去吗？**
答：这个项目主打“本地优先（local-first）”，意思是可以**完全在你的电脑上跑**，成绩也记在你自己的电脑里（一个本地小数据库），不一定要传到网上。

**问：成绩记哪儿了？**
答：记在电脑里一个叫 `event_log` 的“成绩单表格”里。每次考试都会留下：谁考的、考的什么、花了多少、最后判定及格还是不及格。
